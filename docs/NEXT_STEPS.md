## 🚀 Что нужно сделать СЕЙЧАС

### 1. Получите Claude API Key (5 минут)

1. Перейдите на [Anthropic Console](https://console.anthropic.com/)
2. Зарегистрируйтесь или войдите
3. Создайте API Key
4. Скопируйте ключ (начинается с `sk-ant-...`)

### 2. Получите GitHub App Private Key (2 минуты)

1. Откройте: https://github.com/settings/apps/mergeblocker
2. Перейдите на вкладку **"General"**
3. Прокрутите до раздела **"Private keys"**
4. Нажмите **"Generate a private key"**
5. Скачается файл `.pem`
6. Переименуйте его в `private-key.pem`
7. Переместите в корень проекта: `/Users/timtim2379/Desktop/work_dir/py_scripts/merge-bloker/private-key.pem`

### 3. Создайте .env файл (2 минуты)

```bash
cd /Users/timtim2379/Desktop/work_dir/py_scripts/merge-bloker
cp .env.example .env
```

Откройте `.env` и заполните:

```env
GITHUB_APP_ID=2469635
GITHUB_PRIVATE_KEY_PATH=./private-key.pem
GITHUB_WEBHOOK_SECRET=<сгенерируйте: openssl rand -hex 32>
ANTHROPIC_API_KEY=<ваш ключ из шага 1>
```

Для генерации webhook secret:
```bash
openssl rand -hex 32
```

### 4. Установите зависимости (1 минута)

```bash
cd /Users/timtim2379/Desktop/work_dir/py_scripts/merge-bloker
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 5. Запустите сервер (30 секунд)

```bash
python app.py
```

Вы должны увидеть:
```
INFO - Configuration validated successfully
INFO - Starting MergeBlocker on 0.0.0.0:8000
```

### 6. Настройте публичный endpoint (2 минуты)

#### Установите ngrok:

**macOS:**
```bash
brew install ngrok
```

**Другие ОС:**
Скачайте с https://ngrok.com/download

#### Запустите ngrok (в новом терминале):

```bash
ngrok http 8000
```

Вы увидите URL типа: `https://abc123.ngrok.io`

**Скопируйте этот URL!**

### 7. Настройте GitHub App Webhook (1 минута)

1. Откройте: https://github.com/settings/apps/mergeblocker
2. Перейдите на вкладку **"General"**
3. Прокрутите до раздела **"Webhook"**
4. Заполните:
   - **Webhook URL**: `https://abc123.ngrok.io/webhook` (ваш ngrok URL)
   - **Secret**: тот же что в `.env` (из шага 3)
   - **Active**: ✅ включите галочку
5. Нажмите **"Save changes"**

### 8. Настройте Permissions (если еще не сделано) (1 минута)

1. Перейдите на вкладку **"Permissions & events"**
2. Установите:
   - **Pull requests**: `Read & write`
   - **Contents**: `Read-only`
   - **Checks**: `Read & write`
   - **Commit statuses**: `Read & write`
3. В разделе **"Subscribe to events"** отметьте:
   - ✅ **Pull request**
4. Нажмите **"Save changes"**

### 9. Установите App в репозиторий (1 минута)

1. Откройте: https://github.com/settings/apps/mergeblocker/installations
2. Нажмите **"Install"** или **"Configure"**
3. Выберите репозиторий для тестирования
4. Нажмите **"Install"** или **"Save"**

### 10. Создайте тестовый PR (2 минуты)

В установленном репозитории:

```bash
git checkout -b test-mergeblocker
echo "# Testing AI Review" >> README.md
git add README.md
git commit -m "test: Testing MergeBlocker AI review"
git push origin test-mergeblocker
```

Создайте Pull Request на GitHub.

### 11. Проверьте результат! 🎉

Через 30-60 секунд вы должны увидеть:

- ✅ **Check Run**: "AI Code Review - ✅ Review Complete"
- 💬 **Комментарий** от вашего бота с детальным review
- 💬 **Inline комментарии** (если есть замечания к коду)

## 🐛 Если что-то не работает

### Webhook не приходит?

**Проверьте:**
1. Сервер запущен: `curl http://localhost:8000/`
2. ngrok работает: откройте http://127.0.0.1:4040
3. Webhook URL правильный в GitHub App
4. Webhook Active ✅

**Посмотрите Recent Deliveries:**
1. GitHub App → Advanced → Recent Deliveries
2. Найдите последний webhook
3. Посмотрите Response (должен быть 200)

### "Invalid signature" ошибка?

**Решение:**
1. Проверьте `GITHUB_WEBHOOK_SECRET` в `.env`
2. Убедитесь что он совпадает с Secret в GitHub App
3. Перезапустите сервер: `Ctrl+C` и `python app.py`

### "Private key not found" ошибка?

**Решение:**
1. Проверьте что файл существует:
   ```bash
   ls -la private-key.pem
   ```
2. Убедитесь что путь правильный в `.env`:
   ```
   GITHUB_PRIVATE_KEY_PATH=./private-key.pem
   ```

### Review не появляется?

**Проверьте:**
1. Логи сервера на ошибки
2. PR не в draft состоянии
3. Права установлены правильно (Pull requests: Read & Write)
4. Claude API key валидный и есть баланс

### Claude API ошибка?

**Решение:**
1. Проверьте API key: https://console.anthropic.com/
2. Убедитесь что есть кредиты на балансе
3. Проверьте квоты (rate limits)
